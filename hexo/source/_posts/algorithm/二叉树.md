# 二叉树、B 树、B+ 树、红黑树


# 一、二叉树

## 1、 什么是二叉树？
二叉树是一种树形数据结构，每个节点最多有 两个子节点：
  * 左子节点（left）
  * 右子节点（right）

二叉树的结构特点：
  * 根节点（root）：树的顶端节点
  * 叶子节点（leaf）：没有子节点的节点
  * 深度（depth/height）：树从根到最远叶子的层数.


## 2、 二叉树的类型
1. 普通二叉树：没有额外约束，节点左右子节点随意。
2. 满二叉树（Full Binary Tree）：每个节点要么有 0 个子节点，要么有 2 个子节点。
3. 完全二叉树（Complete Binary Tree）：除了最后一层，其他层节点都满，最后一层节点靠左排列。
4. 二叉搜索树 (BST)
    * 左子树所有节点 ≤ 根节点    
    * 右子树所有节点 ≥ 根节点    
    * BST 支持高效查找、插入、删除
5. 平衡二叉树（AVL、红黑树）：BST 的升级版，保证高度平衡，使查找效率稳定在 O(log n)。


## 3、 二叉树的常用操作
1. 遍历（Traversal）
    * 前序遍历：根 → 左 → 右
    * 中序遍历：左 → 根 → 右
    * 后序遍历：左 → 右 → 根
    * 层序遍历（BFS）：按层从上到下、从左到右
2. 查找
    * 普通二叉树：需要遍历所有节点 → O(n)
    * 二叉搜索树：按照 BST 规则 → 平衡树 O(log n)
3. 插入/删除
    * BST 可动态插入/删除，同时保持 BST 约束


## 4、 二叉搜索树（BST）
### 4.1、按照 BST 的规则构建一个二叉树
```python
#  二叉树节点的定义。
class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

#  构建树
# 插入 BST 节点
def insert(root, val):
    if root is None:
        return TreeNode(val)
    if val < root.val:
        root.left = insert(root.left, val)
    else:
        root.right = insert(root.right, val)
    return root

# 构建 BST
root = None
for v in [8, 3, 10, 1, 6, 14, 4, 7, 13]:
    root = insert(root, v)
```
**树结构如下：**
```
        8
       / \
      3   10
     / \     \
    1   6     14
       / \    /
      4   7  13
```

⚠️ <font color='red'>致命问题：</font>
如果数据有序插入：
```
1 → 2 → 3 → 4 → 5
```

会变成：
```
1
 \
  2
   \
    3
     \
      4
```
退化成链表，复杂度变为 O(n)

### 4.2、操作
#### 四种遍历模式：
```python
# 前序遍历
def preorder(node):
    if node:
        print(node.val, end=' ')
        preorder(node.left)
        preorder(node.right)

# 中序遍历
def inorder(node):
    if node:
        inorder(node.left)
        print(node.val, end=' ')
        inorder(node.right)

# 后序遍历
def postorder(node):
    if node:
        postorder(node.left)
        postorder(node.right)
        print(node.val, end=' ')

# 层序遍历
from collections import deque
def level_order(root):
    if not root:
        return
    q = deque([root])
    while q:
        node = q.popleft()
        print(node.val, end=' ')
        if node.left:
            q.append(node.left)
        if node.right:
            q.append(node.right)
```
| 遍历方式                    | 遍历顺序      | 时间复杂度 | 空间复杂度            | 说明               |
| ----------------------- | --------- | ----- | ---------------- | ---------------- |
| 前序遍历 (Preorder)         | 根 → 左 → 右 | O(n)  | O(h) 递归栈，h = 树高度 | 用于复制树或表达式树求值     |
| 中序遍历 (Inorder)          | 左 → 根 → 右 | O(n)  | O(h)             | BST 中序遍历可以得到升序序列 |
| 后序遍历 (Postorder)        | 左 → 右 → 根 | O(n)  | O(h)             | 删除树节点或计算表达式树     |
| 层序遍历 (Level-order, BFS) | 按层从上到下    | O(n)  | O(w) 队列，w = 最大宽度 | 广度优先搜索           |

💡 注：
n = 节点总数
h = 树高度（平衡 BST h ≈ log n，退化链表 h ≈ n）
w = 树最大宽度（最底层节点数，最多 n/2）

#### 查找、插入、删除

```python
# 搜索
def search(root, target):
    if not root:
        return None
    if root.val == target:
        return root
    elif target < root.val:
        return search(root.left, target)
    else:
        return search(root.right, target)
    
# 插入新节点
def insert(root, val):
    if not root:
        return TreeNode(val)
    if val < root.val:
        root.left = insert(root.left, val)
    else:
        root.right = insert(root.right, val)
    return root

# 删除节点
# 删除分三种情况：
# 叶子节点 → 直接删除
# 单子节点节点 → 用子节点替代
# 双子节点节点 → 找右子树最小值（或左子树最大值）替代，然后删除该节点
def delete(root, val):
    if not root:
        return None
    if val < root.val:
        root.left = delete(root.left, val)
    elif val > root.val:
        root.right = delete(root.right, val)
    else:
        # 找到节点
        if not root.left:
            return root.right
        elif not root.right:
            return root.left
        # 双子节点
        min_larger_node = root.right
        while min_larger_node.left:
            min_larger_node = min_larger_node.left
        root.val = min_larger_node.val
        root.right = delete(root.right, min_larger_node.val)
    return root
```
| 操作 | 时间复杂度（平均/平衡） | 最坏情况（退化链表） | 空间复杂度 |
| -- | ------------ | ---------- | ----- |
| 查找 | O(log n)     | O(n)       | O(h)  |
| 插入 | O(log n)     | O(n)       | O(h)  |
| 删除 | O(log n)     | O(n)       | O(h)  |

💡 小结：
四种遍历都是 O(n)，关键在于空间消耗（递归栈或队列）
BST 查找/插入/删除依赖树高度 h，平衡树高度 O(log n)，操作高效

## 5、 平衡二叉搜索树———AVL 树
### 5.1、AVL 树的概念

* **AVL 树**是世界上最早提出的平衡二叉搜索树，由 **G.M. Adelson-Velsky 和 E.M. Landis（1962）** 提出。
* 它是在 **二叉搜索树（BST）** 的基础上增加 **平衡条件**：
  * 每个节点的左右子树高度差 ≤ 1
* 这个高度差称为 **平衡因子 (Balance Factor, BF)**：
    * BF=高度(左子树)−高度(右子树)
* 节点 BF ∈ {-1, 0, 1} → 满足 AVL 条件
* 插入或删除操作可能破坏平衡，需要**旋转**恢复。

### 5.2、AVL 树的特点

1. **严格平衡**
   * 相比普通 BST，高度更小，查找效率高
   * 平衡高度：h ≈ 1.44 × log₂(n + 2) − 0.328（比普通 BST 高度更低）
2. **旋转操作恢复平衡**
   * 当插入/删除破坏 BF 范围时，需要旋转调整：
     1. **单旋转**：
        * LL（左-左）→ 右旋
        * RR（右-右）→ 左旋
     2. **双旋转**：
        * LR（左-右）→ 左旋 + 右旋
        * RL（右-左）→ 右旋 + 左旋
   * 旋转示意图:![旋转示意图](https://raw.githubusercontent.com/cmeng-CM/image-hosting/master/img/algorithm/Tree_Rebalancing.png)  
        >作者 Mhss at zh.wikipedia，CC BY-SA 3.0，https://commons.wikimedia.org/w/index.php?curid=12803426
3. **查找效率高**
   * 平衡树高度保证 O(log n) 查找、插入、删除

# 三、红黑树（解决“二叉树退化问题”）

## 1. 本质

> 红黑树 = 自平衡二叉搜索树

通过：

* 颜色规则
* 旋转操作

来保证树不会变“太高”

---

## 2. 核心规则（简化理解）

1. 红节点不能连续
2. 每条路径黑节点数量一致

---

## 3. 平衡手段

当树不平衡时：

* 变色（轻量调整）
* 旋转（结构调整）

---

## 4. 特点总结

| 特性   | 描述       |
| ---- | -------- |
| 结构   | 二叉树      |
| 高度   | O(log n) |
| 更新   | 快        |
| 使用场景 | 内存数据结构   |

---

## 5. 局限性

👉 树仍然“比较高”（log n）

在磁盘场景下：

* 每一层 = 一次 IO
* 成本很高

# 四、B 树（为磁盘而生）

## 1. 核心思想

> 一个节点存多个 key → 减少树高度

---

## 2. 结构（m 阶 B 树）

* 每个节点最多 m 个子节点
* 每个节点最多 m-1 个 key

```
        [10 | 20]
       /    |    \
   [5,8] [12,15] [25,30]
```

---

## 3. 为什么不会退化成链表？

关键机制：

> **节点满了就分裂**

---

## 4. 插入核心流程

### 当节点未满

直接插入

---

### 当节点满（溢出）

例如：

```
[1,2,3,4]
```

处理：

1. 找中间值（median）
2. 中间值上提
3. 左右分裂

```
      [2]
     /   \
   [1]   [3,4]
```

---

## 5. 核心特性

* 所有叶子节点在同一层（绝对平衡）
* 分裂保证结构稳定
* 高度增长极慢

---

## 6. 为什么是“矮胖型”？

假设：

* 每节点 100 key
* 1 亿数据

高度：

* 第1层：100
* 第2层：1万
* 第3层：100万
* 第4层：1亿

👉 只需要 4 层！

---

# 五、B+ 树（数据库的终极选择）

## 1. 在 B 树基础上的改进

### 核心变化：

| 特性    | B 树        | B+ 树   |
| ----- | ---------- | ------ |
| 数据存储  | 所有节点       | 只在叶子   |
| 非叶子节点 | key + data | 只存 key |
| 叶子节点  | 无连接        | 链表连接   |

---

## 2. B+ 树结构

```
        [10 | 20]
       /    |    \
   [1,5] [10,15] [20,25]
        ↓
   所有叶子节点相连
```

---

## 3. 为什么数据库用 B+ 树？

### ① 更低的高度

非叶子节点不存数据 → 能放更多 key

👉 树更矮

---

### ② 查询路径稳定

所有查询都走到叶子节点

👉 性能一致

---

### ③ 范围查询极强

叶子节点是链表：

```
1 → 2 → 3 → 4 → 5
```

👉 BETWEEN / ORDER BY 非常快

---

### ④ 磁盘 IO 少

* 一层 = 一次 IO
* 通常 3~4 层

👉 查询只需 3~4 次 IO

# 六、MySQL 索引底层（InnoDB）

## 1. 聚簇索引（主键）

```
叶子节点存完整数据：
[id, name, age]
```

特点：

* 数据和索引在一起
* 查询快

---

## 2. 二级索引

```
[name → id]
```

查询流程：

1. 查 name 索引
2. 拿到 id
3. 回主键索引查数据

👉 这就是“回表”

# 七、关键问题总结

## 1. 为什么提中间值？

> 保证分裂后左右尽量均匀 → 保持平衡

---

## 2. 根节点满了怎么办？

* 分裂
* 中间值成为新根
* 树高度 +1

---

## 3. 为什么不会一直变高？

* 分裂是“向上扩展”
* 不是一直向下

---

## 4. 为什么 B+ 树优于红黑树？

| 对比   | 红黑树 | B+ 树 |
| ---- | --- | ---- |
| 高度   | 高   | 很低   |
| IO   | 多   | 少    |
| 范围查询 | 差   | 强    |
| 场景   | 内存  | 磁盘   |

---

# 八、最终总结（面试高分版）

> 二叉搜索树在顺序插入时会退化成链表，因此引入红黑树通过旋转和颜色约束保持平衡；但红黑树仍然是二叉结构，高度较高，不适合磁盘存储。
> B 树通过让每个节点存储多个 key，并在节点溢出时进行分裂，从而降低树的高度；B+ 树在此基础上进一步优化，将数据全部放在叶子节点，并通过链表连接叶子节点，使得范围查询和磁盘 IO 性能更优，因此成为数据库索引的主流结构。
